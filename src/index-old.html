Here are some of the details available to you server-side and/or client-side, once you enable Account Shield:
  • perm_identity User
    User seen on site Boolean flag, to indicate whether that user has visited the site before.
    User seen on page Boolean flag, to indicate whether that user has visited the page before.
    User last seen Date when this user last accessed this site.
    User's devices Array of devices associated with the user.
    User's connections Array of connections associated with the user.
  • devices Device
    Device seen on site Boolean flag, to indicate whether that device has visited the site before.
    Device seen on page Boolean flag, to indicate whether that device has visited the page before.
    Device last seen Date when this device last accessed this site.
    Device type One of 'Mobile', 'Phablet', 'Tablet', 'Laptop' or 'Desktop'.
    Device metrics A host of properties describing the device, including: platform, browser, languages, screen dimensions, device orientation, etc.
  • wifi_tethering Connection
    IP Address Text string, providing the public IP address used by the connection.
    Connection seen on site Boolean flag, to indicate whether that connection has been used to visit the site before.
    Connection seen on page Boolean flag, to indicate whether that connection has been used to visit the page before.
    Connection last seen Date when this connection was last used to access this site.
    Connection Type One of 'Fixed', or 'Mobile'.
  • place Location
    Location co-ordinates Comma separated list of a latitude and longitude position of the city nearest to the location of the device.
    Country code A two character ISO code indicating the country where the device is located.
    Region A text string indicating the region of the country where the device is located.
    City A text string indicating the city nearest to the location of the device.

flash_on Quick to setup, simple to use

Add a couple of lines in the <head> tag of your web pages, and you're good to go.

If you want to tie devices and connections to users, you will also need to make a simple server-side call to Account Shield when you have identified a user (usually at login).

We provide full code examples to get you up and running in next to no time, and even provide plug-ins that do the work for you, for popular CMS and shopping cart systems like Wordpress, Joomla, and Magneto.

What are you waiting for?

Client-side Markup

With this option, Account Shield can add the details you want as attributes to the <body> tag of your page. This works well if you wish to use CSS to show/hide content based on factors like whether the user is a repeat visitor.

Client-side Javascript

Choose this option and Account Shield will create a Javascript object on your page, providing access to all the user, device and connection details. This is the most powerful client-side option, allowing you, through code, to make any aspect of your site responsive to the user's details.

Server-side REST calls

The preferred and most secure option is to call Account Shield from your server-side code. This is easily achieved through a simple to use REST interface.

A word about privacy

We are not the NSA or GCHQ, nor do we approve the tracking of peoples' online activity. In fact one of our greatest tech heroes is Edward Snowden, and we stand with him, the EFF, and many other like-minded people, on the side of defending civil liberties in the digital world.

While the capabilities provided by Account Shield are very powerful, we have deliberately blocked their use across domains - the user, device, and connection IDs that we generate are unique to each domain that uses Account Shield. Unlike other anti-fraud companies, we do not maintain a global database of devices, neither do we share information between our clients.

We do, however, believe that each company has the right to monitor its online visitors, just as a physical store might have security guards or CCTV in its premises. For our customers, we recommend that they place an appropriate disclaimer in the footer of their website.

We believe that our service strikes an appropriate balance in this area, but we welcome contributions to the debate in our forum here.
