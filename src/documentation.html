Getting Started

Download

To start using Account Shield all you have to do is download one of the options below. This is the standard version that comes with both the minified and unminified CSS and JavaScript files.

Downloadfile_download

CDN

You can link to our hosted version of these files using the AWS CDN.


  <!-- Compiled and minified CSS -->
  <link rel="stylesheet" href="https://.../as-client.min.css">

  <!-- Compiled and minified JavaScript -->
  <script src="https://.../as-client.min.js"></script>

NPM

You can also get the latest release using NPM. This release contains both source and minified CSS and JavaScript files.


  npm install as-client

Bower

You can also get the latest release using Bower. This release contains both source and minified CSS and JavaScript files.


  bower install as-client

Client-side CSS

Simple client-side sensing using CSS

Go beyond normal responsive design, to present content or preselect options depending upon device factors such as platform (e.g. Windows, Linux, etc...), browser or device type (e.g. Laptop, Desktop, etc...).

Account Shield provides a simple class expression format which allows for showing/hiding content based upon

Setup

Using Client-side CSS is simple as 1, 2, 3.

  1. Switch on the option in the Dashboard
  2. Inlcude the CSS and JS files in your document
  3. Use the special @ commands in your markup as shown below
How it works

Use Account Shield's special @ commands in your class strings.

@(PROPERTY)
Inserts content of PROPERTY into element <span class="@(device-platform)"><span> would produce markup like <span>Windows<span>
@if(PROPERTY,CLASS)
Enables CLASS if PROPERTY is true <span class="@if(device-platform-linux,penguin)">Lorum ipsum . . .<span> would produce markup like <span class="penguin">Lorum ipsum . . .<span>
@show-if(PROPERTY) /
@visible-if(PROPERTY)
Shows content of element if PROPERTY is true <span class="@show-if(device-platform-windows)">Lorum ipsum . . .<span> (@show accomplishes this using display:none/block, @visible using visibility:hidden/visible - the difference being that @visible reserves the space on the page for the content whether showing or not, @show will collapse the space on the page when not showing)
@hide-if(PROPERTY) /
@invisible-if(PROPERTY)
Hides content of element if PROPERTY is true <span class="@hide-if(device-platform-windows)">Lorum ipsum . . .<span> (the difference between @hide and @invisible follows that for @show/@visible above)

warning Take care not to use any spaces within your @ commands.

You can, however, combine @ commands with other class names, for example: <span class="foo @hide-if(device-platform-windows) bar">

Browser Support

Account shield's Client-side CSS technology relies on CSS3 and JS support, and should work in all except the most ancient and obsolete browsers (i.e. IE6 and Safari 3.1).

Account shield has been designed to gather data without raising notifications or warnings in the end-user's browser. You can use the technology without worrying that your user's will be put off by permission requests to know their location, for example.
